불변성의 중요성

❌ 나쁜 예: 직접 수정

const [user, setUser] = useState({
  name: "찬규",
  age: 28
});

// 참조가 동일 → 변화 감지 실패
user.age = 26;
setUser(user);

✅ 좋은 예: 새 객체 생성

const [user, setUser] = useState({ 
  name: "찬규",
  age: 28 
});

// 새 참조 → 리렌더링
setUser({ ...user, age: 26 });

💡 불변성(Immutability)이란?

데이터를 직접 수정하지 않고, 변경이 필요하면 새로운 데이터를 만드는 원칙입니다.
React는 얕은 비교(===)로 상태 변화를 감지하므로, 객체의 참조가 바뀌어야 리렌더링이 발생합니다.
